You can set git config to rebase whenever pulling using git config pull.rebase true.

But then you can‚Äôt pull without first committing your changes. This is a pain if you don‚Äôt want to or can‚Äôt keep your working directory clean.

To deal with this you can set another git config setting to stash whenever rebasing git config rebase.autoStash true. This will stash your changes before pulling, and then reapply them after the pull is complete.

Set per repo

Now I can pull and rebase easily without having to commit my changes, and my git history is clean and simple by setting these two config options on each repo:

git config pull.rebase true
  git config rebase.autoStash true
